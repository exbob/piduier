# 前端代码结构设计

## 方案对比

### 方案 A：单文件方案（`web/index.html`）

将所有 HTML、CSS、JavaScript 代码都放在一个 `index.html` 文件中。

#### 优点
1. **简单直接**
   - 只需要一个文件，部署方便
   - 不需要管理多个文件的依赖关系
   - 适合小型项目或原型开发

2. **无依赖问题**
   - 不需要考虑文件加载顺序
   - 减少 HTTP 请求（虽然可以合并，但单文件本身就是合并的）

3. **易于分享**
   - 单个文件便于分享和测试

#### 缺点
1. **可维护性差**
   - 文件会变得非常大（预计 2000+ 行）
   - HTML、CSS、JavaScript 混在一起，难以阅读和维护
   - 代码高亮和语法检查效果差

2. **不利于协作**
   - 多人协作时容易产生冲突
   - 难以进行代码审查

3. **不利于复用**
   - 无法复用 CSS 样式
   - 无法复用 JavaScript 函数
   - 难以模块化

4. **性能考虑**
   - 虽然单文件减少了 HTTP 请求，但：
     - 浏览器无法并行加载资源
     - 无法利用浏览器缓存（CSS/JS 变更需要重新加载整个 HTML）
     - 无法按需加载

### 方案 B：分离文件方案（推荐）

将 HTML、CSS、JavaScript 分离到不同文件，保持简单但结构清晰。

```
web/
├── index.html          # HTML 结构
├── css/
│   └── style.css       # 所有样式
└── js/
    └── app.js          # 所有 JavaScript 逻辑
```

#### 优点
1. **可维护性好**
   - 代码分离，职责清晰
   - 易于阅读和修改
   - 语法高亮和代码检查效果好

2. **性能优化**
   - 浏览器可以并行加载 CSS 和 JS
   - 可以单独缓存 CSS 和 JS 文件
   - 可以按需加载（如果未来需要）

3. **便于扩展**
   - 如果未来需要，可以进一步拆分（如按模块拆分 JS）
   - 可以引入构建工具（如打包、压缩）

4. **符合最佳实践**
   - 符合 Web 开发标准
   - 便于团队协作

#### 缺点
1. **文件数量增加**
   - 需要管理 3 个文件（但结构清晰）

2. **需要确保文件路径正确**
   - 相对路径需要正确配置

## 推荐方案

### 推荐：**方案 B（分离文件方案）**

#### 理由
1. **项目规模**
   - 这是一个完整的监控系统，功能较多
   - 预计最终代码量：HTML ~500 行，CSS ~800 行，JavaScript ~1500 行
   - 单文件会超过 2500 行，难以维护

2. **可维护性**
   - 分离后每个文件职责清晰，易于维护
   - 符合 Web 开发最佳实践

3. **性能**
   - 浏览器可以并行加载资源
   - 可以单独缓存，提高性能

4. **扩展性**
   - 如果未来需要，可以进一步模块化
   - 可以引入构建工具优化

### 文件结构

```
web/
├── index.html          # HTML 结构（~500 行）
├── css/
│   └── style.css       # 所有样式（~800 行）
└── js/
    └── app.js          # 所有 JavaScript 逻辑（~1500 行）
```

### 实现建议

#### 1. HTML 结构（`index.html`）
- 只包含 HTML 结构
- 通过 `<link>` 引入 CSS
- 通过 `<script>` 引入 JavaScript
- 保持简洁，不包含内联样式和脚本

#### 2. CSS 样式（`css/style.css`）
- 所有样式集中在一个文件
- 使用 OKLCH 色彩空间
- 响应式布局
- 模块化组织（注释分隔不同模块）

#### 3. JavaScript 逻辑（`js/app.js`）
- 所有逻辑集中在一个文件
- 使用模块化组织（函数分组，注释分隔）
- API 调用、UI 更新、事件处理等

### 如果未来需要进一步拆分

如果项目继续扩展，可以考虑按模块拆分：

```
web/
├── index.html
├── css/
│   ├── base.css        # 基础样式
│   ├── layout.css      # 布局样式
│   └── components.css  # 组件样式
└── js/
    ├── api.js          # API 调用
    ├── ui.js           # UI 更新
    ├── system.js       # 系统监控
    ├── network.js      # 网络配置
    └── hardware.js     # 硬件控制
```

但**当前阶段不需要**，保持简单即可。

## 总结

**推荐使用分离文件方案**（HTML、CSS、JavaScript 分离），因为：
1. ✅ 可维护性好，代码清晰
2. ✅ 性能更好，可以并行加载和缓存
3. ✅ 符合 Web 开发最佳实践
4. ✅ 便于未来扩展

**文件结构**：
- `web/index.html` - HTML 结构
- `web/css/style.css` - 所有样式
- `web/js/app.js` - 所有 JavaScript 逻辑

这样既保持了简单性，又提高了可维护性。
