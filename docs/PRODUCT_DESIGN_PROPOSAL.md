| 版本策略 | 构建时由 `git describe --tags --always --long` 注入 |

## 1. 背景与目标

### 1.1 背景

树莓派 5 的 **40pin 连接器**在嵌入式软硬件开发中常用于 GPIO、PWM、I2C、SPI、UART 等外设调试。现有流程往往依赖命令行与零散工具，效率低、易出错、不利于现场演示与客户验收。

本产品（与仓库内 **piduier** 能力对齐）提供 **Web 控制台**：在树莓派 5 上运行 HTTP 服务，浏览器访问即可查看引脚功能、配置接口并执行读写/收发，服务于 **开发调试与测试** 场景。

### 1.2 产品目标

1. **一目了然**：展示当前 40pin 各引脚的**功能角色**（电源/地/复用功能等），与树莓派 5 常见引脚定义一致。
2. **可配置、可观测**：对支持 GPIO 的引脚可设方向；输入实时显示电平，输出可设高低电平。
3. **波形与总线调试**：PWM 可调周期/占空比并可开关；I2C/SPI 以**主设备**身份配置参数并读写从设备寄存器；UART 可设串口参数并收发数据。
4. **工程化交付**：本地 x86 可启动查看 UI；目标机在树莓派 5（ARM64 Linux）上交叉编译部署；提供**简便的构建与部署**说明（脚本或文档一键化倾向）。

### 1.3 非目标（当前版本明确不做或降级）

- 不作为面向公网的多租户 SaaS；默认假设 **局域网或本机访问**。
- 不承诺替代专业逻辑分析仪/示波器；PWM/波形以**软件可控输出**为主，精度与抖动以硬件与内核为准。
- 多板卡、多用户权限模型可在后续版本扩展；本文档以**单设备、单管理员**为主。

## 2. 用户与场景

### 2.1 用户角色

| 角色 | 诉求 |
|------|------|
| 嵌入式研发 | 快速改 GPIO/PWM/总线参数，看寄存器读写结果与串口日志 |
| 客户/集成方 | 验收「功能是否覆盖需求清单」、界面是否易用、部署是否清晰 |

### 2.2 典型场景

1. 上电后打开浏览器 → 查看 40pin 图 → 将某 GPIO 设为输出高 → 万用表/示波器验证。
2. 从设备接在 I2C/SPI → 设置总线参数 → 读芯片 ID 寄存器 → 写配置寄存器。
3. 外设 UART 调试 → 设置波特率等 → 发送指令、查看接收区 hex/文本。
4. 本地改 UI → x86 起服务看样式 → 交叉编译 → 部署到树莓派 5 实机验证。

## 3. 功能需求

### 3.1 引脚功能展示（40pin）

**需求 ID：F-01**

- 以**符合习惯**的方式展示 40 个物理引脚（例如双列对照表或等效 pinout 视图）。
- 每个引脚至少展示：**物理编号**、**默认/当前功能类型**（电源、GND、GPIO、I2C、SPI、UART、PWM、EEPROM 等）、**文字说明**（与 BCM GPIO 编号等关联说明，与 Pi5 文档一致）。
- 若某引脚因内核/overlay 配置当前不可用，应**明确提示**（与 3.x 各子能力错误信息一致）。

**验收要点**：与官方 40pin 定义无矛盾；客户可对照硬件丝印完成连线。

---

### 3.2 GPIO（方向、输入实时电平、输出电平）

**需求 ID：F-02**

- 对**允许配置为 GPIO**的引脚（以产品实现与安全策略为准，部分引脚可只读展示）：
  - 可设置 **输入 / 输出**。
  - **输入**：以可配置的刷新策略**实时显示**高/低（或 1/0）；需说明刷新间隔与后端实现方式（轮询 vs 事件，由研发在实现说明中落地）。
  - **输出**：可设置 **高电平 / 低电平**（或 1/0）。
- 冲突检测：若引脚被系统占用为其他功能，应返回**可读错误**，前端提示用户检查 `raspi-config`/设备树/驱动占用。

**验收要点**：万用表或逻辑探头可验证输出；输入可接已知电平源验证显示。

---

### 3.3 PWM（周期、占空比、开关）

**需求 ID：F-03**

- 对支持 PWM 的引脚（如硬件 PWM 可用引脚，具体以 Pi5 + Linux 为准）：
  - 可配置 **周期**（或频率）与 **占空比**（范围与单位在 UI 与 API 中统一，如 Hz + %）。
  - 提供 **启用 / 停用** 开关；停用后释放或停止输出（行为在实现章节写明）。
- 参数校验：超出硬件或驱动支持范围时给出明确错误信息。

**验收要点**：示波器或频率计可验证频率与占空比量级正确；开关可切断输出。

---

### 3.4 I2C / SPI（主设备、参数、寄存器读写）

**需求 ID：F-04**

**I2C（主）**

- 可选择总线实例（如 `i2c-1` 等，以设备节点为准）。
- 可配置 **从设备地址**（7bit/10bit 策略与 UI 约定一致）、**时钟速率**（在驱动允许范围内）。
- 支持 **读寄存器 / 写寄存器**（或等价的「写寄存器地址 + 读 N 字节」）；字节长度可配置。
- 可选：简单 **总线扫描**（发现地址列表）作为调试辅助（若纳入范围则在里程碑中排期）。

**SPI（主）**

- 可选择 SPI 设备节点（如 `/dev/spidevX.Y`）。
- 可配置 **模式**（CPOL/CPHA）、**位宽**（若适用）、**最大速度** 等常用参数。
- 支持按「寄存器地址 + 读写长度」或「原始事务」完成与从设备的通信（具体 API 形态由研发在接口文档中定义，需与客户确认是否必须「寄存器语义」还是允许「原始字节」）。

**验收要点**：对接已知从设备（如传感器 demo 板）可稳定读写 ID/配置寄存器。

---

### 3.5 UART（参数、收发）

**需求 ID：F-05**

- 可选择串口设备（如 `/dev/ttyAMAx` 等，以 Pi5 启用的串口为准）。
- 可配置 **波特率、数据位、校验、停止位、流控**（实现范围以系统 termios 能力为准）。
- 提供 **发送区**（文本/Hex 可选其一或两者）与 **接收区**（可清屏、可暂停或滚动策略）。
- 连接占用：打开串口时应处理「已被其他进程占用」的错误。

**验收要点**：与 USB 转串口对传或回环（若硬件允许）可互通数据。

## 4. 非功能需求

### 4.1 平台与构建

| 项 | 要求 |
|----|------|
| 本地开发机 | x86_64，可编译并启动以**预览 Web UI**（可用 mock/桩服务或与后端同进程，由实现选定并在构建文档说明） |
| 目标运行环境 | 树莓派 5，ARM64 Linux（与项目当前支持一致） |
| 交叉编译 | 提供可重复的 CMake/脚本流程；文档中写明工具链与依赖 |
| 版本号 | 构建产物中展示的版本字符串来自 `git describe --tags --always --long` |

### 4.2 性能与实时性

- GPIO 输入「实时」为**尽力而为**：默认刷新间隔可配置（如 100ms～500ms），并在 UI 标明「非硬实时」。
- 高频 SPI/I2C 压测可作为后续性能项，首版以**功能正确**与**错误可诊断**优先。

### 4.3 安全与权限

- 服务默认**仅监听局域网**或本机（具体 bind 地址在部署文档写明）。
- 硬件访问依赖 Linux 权限（如 `gpiochip`、`spidev`、`i2c`、`tty` 等）；与 README 中 **piduier 组 + udev** 策略对齐。
- **不在仓库文档中保存 SSH 密码**；部署使用 **SSH 公钥** 或部署时手动输入。示例主机可写：`lsc@172.16.1.101`（密码由操作者本地保管）。

### 4.4 可维护性

- REST（或等价）API 与 Web 前端字段命名一致；错误码与错误信息可被客户截图反馈。
- 功能模块（GPIO / PWM / I2C / SPI / UART）在代码与文档上边界清晰，便于分阶段交付。

## 5. 信息架构与体验设计（概要）

### 5.1 导航结构（建议）

- **总览 / 40pin**：pinout + 图例 + 与当前页签联动。
- **GPIO**：可配置引脚列表或嵌入在 pinout 中（与现有实现路线一致即可）。
- **PWM**：独立页或折叠面板，仅展示可用 PWM 引脚。
- **I2C**：总线选择、参数、扫描（可选）、寄存器读写表单。
- **SPI**：设备选择、参数、事务/寄存器读写表单。
- **UART**：设备选择、参数、收发终端式界面。
- **系统/关于**：版本号（`git describe`）、依赖接口启用提示、文档链接。

### 5.2 关键交互原则

- **危险操作**（若后续加入重启/关机等）需二次确认；纯调试场景下的 GPIO 输出变化建议有简短提示。
- **错误可行动**：提示用户检查接口是否启用、权限、设备占用、参数范围。
- **移动端**：窄屏下 pinout 可横向滚动或改为分段卡片（与第二轮前端优化路线一致）。

## 6. 接口与数据（研发参考）

> 以下为逻辑接口清单，具体路径与方法以 `router`/OpenAPI 或 `docs` 中接口说明为准。

| 领域 | 能力 | 说明 |
|------|------|------|
| 系统 | 版本信息 | 返回 `git describe` 注入的版本字符串 |
| GPIO | 列表/单脚状态 | 模式、值、是否可配 |
| GPIO | 设模式/设值 | POST，错误返回明确原因 |
| PWM | 查询/配置/开关 | 与 Linux pwm 栈一致 |
| I2C | 读写 | 地址、寄存器、长度、数据 |
| SPI | 传输 | 模式、速率、写入/读出缓冲 |
| UART | 打开参数/发送/接收缓冲 | WebSocket 或轮询二选一，需在实现设计中拍板 |

## 7. 构建、部署与验收环境

### 7.1 本地（x86）

- 目的：快速验证 **HTML/CSS/JS** 与（可选）本地桩 API。

- 示例目标：`lsc@172.16.1.101`（局域网）。

## 9. 风险与依赖

- **内核/设备树**：引脚复用与 PWM/SPI 设备节点随系统配置变化，需在产品中集中说明「先启用接口」。
- **权限**：无 udev/组配置时功能不可用，需在首次启动或「关于」页检测并提示。
- **实时性**：浏览器轮询与内核调度限制「实时」语义，需在 PRD 层与客户对齐预期。
- **安全**：局域网暴露 HTTP 控制硬件存在误操作与未授权访问风险，后续可增认证或仅绑定 127.0.0.1。
